# Instrument Studio Minimal Interactive

Use for browser-native instruments, creative tools, audio/visual toys, focused interactive demos, or one-screen experimental apps that need a minimal dark stage, central control object, tiny instructions, and no marketing-page clutter.

## Reusable strengths
- single_screen_stage
- central_control
- minimal_microcopy
- interactive_focus
- dark_studio_palette

## Pattern rules
- The first screen is the product: a centered interactive object, quiet brand lockup, and only the controls needed to begin.
- Interaction hints are compact and below the focal control; they should not become onboarding prose.
- If the target app is not implemented, do not fake functionality. Use disabled states or clearly marked preview controls.
- Mobile keeps the control object within thumb-friendly reach and resists adding extra marketing sections.

## Anti-copy
- Do not reuse donor brand names, exact headlines, exact facts, exact product names, exact numbers, or exact visual diagrams.
- Use this example for layout grammar and interaction/state grammar only.
