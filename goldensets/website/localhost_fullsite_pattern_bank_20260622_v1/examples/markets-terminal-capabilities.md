# Markets Terminal Blue Dashboard

Use for finance, analytics, trading, market-data, observability, or operations products that need a dense live-terminal landing page with tables, charts, tickers, watchlists, and a crisp product-console hero.

## Reusable strengths
- product_console_hero
- ticker_strip
- data_tables
- dense_cards
- blue_dark_contrast

## Pattern rules
- Hero copy is subordinate to the live product surface; the visual proof is the terminal itself, not a generic illustration.
- A ticker/status strip creates continuity between hero and content, then dense capability cards explain each panel’s job.
- Charts, order books, watchlists, and tables should use real supplied data or neutral placeholders clearly marked as sample.
- Mobile keeps table readability by stacking market groups and shortening surrounding prose, not by hiding the product surface.

## Anti-copy
- Do not reuse donor brand names, exact headlines, exact facts, exact product names, exact numbers, or exact visual diagrams.
- Use this example for layout grammar and interaction/state grammar only.
