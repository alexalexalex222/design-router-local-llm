# Developer Docs Quickstart Dark

Use for developer-documentation, SDK, CLI, database, API, or infrastructure quickstart pages that need a navigable docs shell, right-hand page outline, code blocks, copy affordances, callouts, and footer taxonomy.

## Reusable strengths
- docs_layout
- code_blocks
- toc_navigation
- mobile_docs_tabs
- technical_callouts

## Pattern rules
- The document is the product: a compact header, left nav, content column, and right outline frame the actual steps.
- Each section pairs a short explanation with executable-looking code blocks and copy affordances.
- Callouts carry operational caveats; do not invent commands, schema names, package names, or licensing claims.
- Mobile swaps side rails for compact topic chips and a menu while preserving readable code-block wrapping.

## Anti-copy
- Do not reuse donor brand names, exact headlines, exact facts, exact product names, exact numbers, or exact visual diagrams.
- Use this example for layout grammar and interaction/state grammar only.
